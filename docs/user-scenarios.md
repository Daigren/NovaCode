# Пользовательские сценарии (CodeSphere)

## Сценарий 1: Путь Гостя (Регистрация перед покупкой)
**Цель:** Изучить курсы и создать аккаунт для дальнейшей записи.
1. Гость заходит на **Главную страницу** и читает о платформе.
2. Через навигационное меню переходит в **Каталог курсов**.
3. Используя фильтр (или визуальный поиск), находит курс «Backend-разработчик на PHP».
4. Кликает на карточку и переходит на **Детальную страницу курса**.
5. Нажимает кнопку «Записаться». Так как он не авторизован, система перенаправляет его на страницу **Авторизации/Регистрации**.
6. Гость заполняет форму регистрации (Имя, Email, Пароль).
7. Данные отправляются через Fetch API. При успешной валидации Гость становится **Пользователем** и перенаправляется в Личный кабинет.

## Сценарий 2: Путь Пользователя (Оформление заявки)
**Цель:** Отправить заявку на курс и проверить её статус.
1. Пользователь заходит на сайт и авторизуется в системе (переходит в **Личный кабинет**).
2. Переходит в **Каталог курсов** и выбирает нужный курс.
3. На **Детальной странице курса** нажимает кнопку «Записаться».
4. JavaScript (Fetch API) отправляет запрос на сервер без перезагрузки страницы. На кнопке появляется индикатор загрузки, а затем сообщение «Заявка успешно отправлена».
5. Пользователь переходит в **Личный кабинет** → раздел **«Мои действия»**.
6. Видит в списке свою новую заявку со статусом `В ожидании`.

## Сценарий 3: Путь Администратора (Обработка заявки)
**Цель:** Проверить новые заявки и одобрить одну из них.
1. Сотрудник вводит свои учетные данные на странице входа. Система распознает роль `admin` и открывает доступ к **Административной панели**.
2. Администратор переходит в раздел **«Управление действиями (Заявки)»**.
3. Видит таблицу со всеми заявками. Находит последнюю заявку (от Пользователя из Сценария 2), которая имеет статус `В ожидании`.
4. Нажимает кнопку «Редактировать статус» рядом с заявкой.
5. В выпадающем списке меняет статус на `Одобрено` и сохраняет изменения (через AJAX-запрос).
6. Заявка обновляется в таблице. (В этот момент у Пользователя в Личном кабинете статус также меняется на «Одобрено»).

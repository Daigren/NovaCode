# Структура сайта проекта CodeSphere

Архитектура веб-приложения разделена на три логических блока в зависимости от прав доступа пользователей. Проект включает 11 основных уникальных экранов.

## 1. Публичная часть (Доступна всем)
Эта часть ориентирована на привлечение пользователей и предоставление информации о курсах.
* **Главная страница (`index.php`)** — презентация платформы, преимущества, популярные направления.
* **Каталог курсов (`catalog.php`)** — полный список образовательных программ с возможностью поиска и фильтрации.
* **Детальная страница курса (`course.php?id=...`)** — подробное описание выбранной программы, цена, программа обучения и кнопка «Записаться».
* **Регистрация (`register.php`)** — форма создания новой учетной записи.
* **Авторизация (`login.php`)** — форма входа в систему для существующих пользователей и администраторов.

## 2. Личный кабинет (Доступен авторизованным пользователям)
Закрытая зона для студентов платформы.
* **Профиль пользователя (`profile.php`)** — панель управления личными данными (имя, email) с возможностью их изменения.
* **Мои заявки (`my-applications.php`)** — страница со списком всех курсов, на которые пользователь оставил отклик, и отображением их текущего статуса (В ожидании / Одобрено / Отклонено).

## 3. Административная часть (Доступна только администратору)
Панель управления платформой и бизнес-процессами.
* **Главный экран администратора (`admin/index.php`)** — дашборд с общей статистикой (количество пользователей, курсов, новых заявок).
* **Управление каталогом (`admin/courses.php`)** — интерфейс для добавления новых курсов, редактирования текущих и их удаления (реализация CRUD).
* **Управление категориями (`admin/categories.php`)** — добавление и удаление направлений обучения (Frontend, Python и т.д.).
* **Управление заявками (`admin/applications.php`)** — таблица всех поступивших заявок от пользователей с функционалом изменения их статусов.
